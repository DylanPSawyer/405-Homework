The HTML Elements I Know

  • script:
    • MDN Script Element
    • Display: It's content won't be displyed, So I guess "display:none"
    • Attributes:
      • async: This is a Boolean attribute indicating that the browser should, if possible, execute the script asynchronously
      • src: This attribute specifies the URI of an external script; this can be used as an alternative to embedding a script directly within a document.
      • defer: This Boolean attribute is set to indicate to a browser that the script is meant to be executed after the document has been parsed, but before firing DOMContentLoaded.
  • style:
    • MDN style Element
    • Display: It's content won't be displyed, So I guess "display:none"
    • Attributes:
      • type: This attribute defines the styling language as a MIME type (charset should not be specified). This attribute is optional and defaults to text/css if it is not specified — there is very little reason to include this in modern web documents.
      • media: This attribute defines which media the style should be applied to. Its value is a media query, which defaults to all if the attribute is missing.
      • title: This attribute specifies alternative style sheet sets.
  • img:
    • MDN img Element
    • display:inline
    • Attributes:
      • width:The intrinsic width of the image in pixels.
      • heigt: The intrinsic height of the image in pixels.
      • src: The image URL.
  • strong:
    • MDN strong Element
    • display:inline
    • Attributes: only the global attributes
  • em:
    • MDN em Element
    • display:inline
    • Attributes: only the global attributes
  • span:
    • MDN span Element

Vocab Definitions

  • HTTP
    • My Definition: Http(Hyper Text Transfer Protocol) is the communication protocol used by most to access the World Wide Web

    • https://developer.mozilla.org/en-US/docs/Glossary/HTTP
      : The HyperText Transfer Protocol (HTTP) is the underlying network protocol that enables transfer of hypermedia documents on the Web, typically between a browser and a server so that humans can read them. The current version of the HTTP specification is called HTTP/2.
  • HTML
    • My Definition: HTML(Hyper Text Markup language) The Markup Langauge created to describe a web page, it is read by browsers

    • https://developer.mozilla.org/en-US/docs/Glossary/HTML
      : HTML (HyperText Markup Language) is a descriptive language that specifies webpage structure.
  • Element
    • My Definition: A piece of the HTML syntax, consists of, Opening Tag, Content, and Closing tag

    • https://developer.mozilla.org/en-US/docs/Glossary/Element
      : An element is a part of a webpage. In XML and HTML, an element may contain a data item or a chunk of text or an image, or perhaps nothing. A typical element includes an opening tag with some attributes, enclosed text content, and a closing tag
  • Tag
    • My Definition: A piece of HTML or XML syntax which wraps the content of an element

    • https://developer.mozilla.org/en-US/docs/Glossary/tag
      In HTML a tag is used for creating an element. The name of an HTML element is the name used in angle brackets such as <p> for paragraph. Note that the end tag's name is preceded by a slash character, </p>, and that in empty elements the end tag is neither required nor allowed. If attributes are not mentioned, default values are used in each case.
  • Block (CSS)
    • My Definition: An HTML document that forces itself to be created on a newline

    • https://developer.mozilla.org/en-US/docs/Glossary/Block/CSS
      A block on a webpage is an HTML element that appears on a new line, i.e. underneath the preceding element in a horizontal writing mode, and above the following element (commonly known as a block-level element). For example, <p> is by default a block-level element, whereas <a> is an inline element — you can put several links next to one another in your HTML source and they will sit on the same line as one another in the rendered output.
  • Doctype
    • My Definition: Doctype specifies what version of HTML an HTML document is using ie:HTML5 vs HTML4.01

    • https://developer.mozilla.org/en-US/docs/Glossary/Doctype
      In HTML, the doctype is the required "<!DOCTYPE html>" preamble found at the top of all documents. Its sole purpose is to prevent a browser from switching into so-called “quirks mode” when rendering a document; that is, the "<!DOCTYPE html>" doctype ensures that the browser makes a best-effort attempt at following the relevant specifications, rather than using a different rendering mode that is incompatible with some specifications.
  • Head
    • My Definition: HTML element before the body element, largley contains un dispalyed meta-data and calls to external stylesheets or scripts

    • https://developer.mozilla.org/en-US/docs/Glossary/head
      The Head is the part of an HTML document that contains metadata about that document, such as author, description, and links to CSS or JavaScript files that should be applied to the HTML.
  • Attribute
    • My Definition: Declared inside the opening tag of an HTML element, specifies behavior of the element for example the "href" in an a element

    • https://developer.mozilla.org/en-US/docs/Glossary/attribute
      An attribute extends a tag, changing its behavior or providing metadata. An attribute always has the form name=value (the attribute's identifier followed by its associated value).
  • Boolean
    • My Definition: A data type repesenting a binary true or false(1 or 0) value, it is used in various programming langauges like Javascript(despite being non-strongly typed), C++, PHP, Java, etc

    • https://developer.mozilla.org/en-US/docs/Glossary/boolean
      In computer science, a Boolean is a logical data type that can have only the values true or false. For example, in JavaScript, Boolean conditionals are often used to decide which sections of code to execute (such as in if statements) or repeat (such as in for loops).
  • DOM
    • My Definition: The Document Object Model, used by Javascript to represent, and modify the HTML elements on the webpage

    • https://developer.mozilla.org/en-US/docs/Glossary/dom
      The DOM (Document Object Model) is an API that represents and interacts with any HTML or XML document. The DOM is a document model loaded in the browser and representing the document as a node tree, where each node represents part of the document (e.g. an element, text string, or comment).
  • CSS
    • My Definition: CSS Cascading Style Sheets, is a syntax that can be read by browsers, it is used to style an html document. It can be placed in a linked external .css file or emmbbed on the page inside a style element

    • https://developer.mozilla.org/en-US/docs/Glossary/css
      CSS (Cascading Style Sheets) is a declarative language that controls how webpages look in the browser. The browser applies CSS style declarations to selected elements to display them properly. A style declaration contains the properties and their values, which determine how a webpage looks.
  • Property (CSS)
    • My Definition: A key-value pair placed inside a CSS Selector which gives some description of how to display the selected element(s) in an HTML file

    • https://developer.mozilla.org/en-US/docs/Glossary/property/css
      A CSS property is a characteristic (like color) whose associated value defines one aspect of how the browser should display the element.
  • Selector (CSS)
    • My Definition: Found in CSS it is a combination of HTML Classes, Element types and HTML ID's used to select a specific element or range of elements in a HTML document, functions sort of like a file path

    • https://developer.mozilla.org/en-US/docs/Glossary/CSS_Selector
      A CSS selector is the part of a CSS rule that describes what elements in a document the rule will match. The matching elements will have the rule's specified style applied to them.
